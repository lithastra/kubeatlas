read-side multi-cluster RBAC visibility, internal GraphStore v2, and

committed; what remains needs a human — credentialed/irreversible
release actions and a live-infra performance run.

steps below (invariant 2.6, and the same policy the v1.4 runbook set).

git add test/verify/perf-baseline-v1.5.json && git commit -s

## 7. Frozen-v1alpha1 invariant assertions (must stay green)

These are the crux of a non-breaking v1.5 (invariant 2.2). `phase5.sh`
checks them; verify by hand if in doubt:

- `curl /api/v1/info | jq -r .graphstore_version` == `"v2"`.
- `curl -s -o /dev/null -w '%{http_code}' /api/v1alpha1/graph?level=cluster` == `200` (not 404).
- No `_sunset` field anywhere in a `v1alpha1` response.
- Served `/api/v1alpha1/openapi.json` path set == committed
  `api/openapi-v1alpha1.json` (no additions/removals; no `/otel/` path).
- `/api/v1alpha1/otel/overlay` → `404` (overlay is v1-only).
- The GraphStore v2 migration deleted no `v1alpha1`-related columns.
- `npm view @lithastra/plugin-kubeatlas@1.0.0` is "not found" at
  main-release time (Backstage publishes last, §6).
